The folder deletion flow is currently broken — the folder gets deleted regardless of whether the user selects "OK" or "Cancel". Fix the following:

"Cancel" must fully abort the deletion and return the user to the folder with no changes made
Replace the current single "OK" confirmation with two clearly distinct options:

"Delete folder only" — removes the folder but keeps all stocks in "My Watchlist" and any other folders they belong to
"Delete folder and remove stocks" — removes the folder and fully removes all its stocks from the Watchlist and all other folders


Neither action should trigger until the user explicitly selects one of these two options — no accidental deletions
